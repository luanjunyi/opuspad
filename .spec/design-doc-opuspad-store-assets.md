# OpusPad Chrome Web Store Assets — System Design Doc

## Overview

Prepare all Chrome Web Store assets for OpusPad: text copy (descriptions, taglines, bullet summaries) and visual assets (screenshots, promo tiles). The text copy is saved to `store_assets/texts.md` for easy copy-paste into the CWS developer dashboard. The visual assets are generated via a Playwright script (`scripts/generate_store_assets.mjs`) and saved to `store_assets/`. Target audience: AI coding power users who review and edit Markdown files generated by Claude Code, Cursor, and similar agents.

```
If you use AI coding agents like Claude Code or Cursor, you constantly deal with generated Markdown: CLAUDE.md, implementation plans, architecture notes, and specs.

Opening these raw files in the browser is an unreadable mess. Opening them in a heavy desktop app just for a quick review is overkill.

OpusPad is the sweet spot. A lightweight Chrome extension that opens your local Markdown files and renders them beautifully.

KEY FEATURES:

• WYSIWYG Editor — Tables, code blocks, and lists render cleanly. Edit text naturally without worrying about Markdown syntax.

• Source Mode — Toggle to raw Markdown with full syntax highlighting for precision edits. A compatibility guard automatically falls back to source mode for complex syntax.

• Directory Sidebar — Browse your project's file tree. Smart fuzzy search lets you jump to any file instantly.

• Auto Save — Your edits persist to disk automatically. No manual save required.

• External Change Detection — When an AI agent updates a file on disk, OpusPad detects it and reloads automatically.

• File Management — Create new Markdown files and delete old ones directly from the sidebar.

• 100% Local & Private — Your files never leave your machine. Zero telemetry, zero cloud sync, zero data collection.

• Lightweight — No Electron overhead. Runs natively inside Chrome. Opens instantly.

PERFECT FOR:
- Reviewing CLAUDE.md and agent rulesets
- Editing architecture plans and specs
- Quick README updates
- Refining any AI-generated documentation

OpusPad is free and open source.
```

## Screenshot Concepts

### Screenshot 1: Review CLAUDE.md
- **Overlay title**: `Render CLAUDE.md beautifully.`
- **Overlay subtitle**: `Instantly review AI agent instructions in a clean, readable format.`
- **What to show**: OpusPad WYSIWYG editor with a realistic `CLAUDE.md` file open. The file should contain:
  - A `# Project Context` heading
  - A bulleted list of "Code Style Rules"
  - A fenced code block showing a terminal command
  - A table with project configuration
- **Sidebar**: Visible, showing a directory tree with files like `CLAUDE.md`, `README.md`, `src/`, `docs/`
- **Benefit**: User sees that raw agent instructions become instantly legible
- **Layout**: Full 1280×800 capture of the extension UI. Add a semi-transparent overlay bar at the top with the title text in white on dark background

### Screenshot 2: WYSIWYG Edit + Save
- **Overlay title**: `Edit specs visually. Save as clean Markdown.`
- **Overlay subtitle**: `Fix typos, reorder sections, and refine AI output naturally.`
- **What to show**: OpusPad WYSIWYG editor with an `implementation-plan.md` file open. Show:
  - A heading: "## Phase 1: Database Migration"
  - A numbered list of implementation steps
  - A code block with SQL
  - Cursor positioned in the text (blinking cursor visible)
- **Sidebar**: Visible
- **Benefit**: User sees that editing feels natural, not like fighting raw Markdown syntax
- **Layout**: Same 1280×800 format with overlay bar

### Screenshot 3: Local & Private
- **Overlay title**: `100% local. Zero uploads.`
- **Overlay subtitle**: `Your proprietary specs never leave your machine.`
- **What to show**: OpusPad with a sensitive-looking file open, e.g., `architecture-notes.md` with:
  - Infrastructure details, API endpoints, database schema references
  - A visual emphasis (callout circle or subtle glow) on the `file://` protocol in a visible path indicator
- **Sidebar**: Visible, showing project-style file tree
- **Benefit**: User immediately understands this is fully local software
- **Layout**: Same format with overlay bar. Consider a lock icon or shield icon in the overlay

### Screenshot 4: Source Mode
- **Overlay title**: `Toggle to raw Markdown instantly.`
- **Overlay subtitle**: `Full control when syntax gets complex.`
- **What to show**: OpusPad in Source Mode (CodeMirror editor). The raw Markdown should show:
  - `## API Specification` heading in Markdown syntax
  - A raw Markdown table (`| Column | Column |`)
  - Bold/italic syntax (`**bold**`, `_italic_`)
  - A fenced code block with language tag
  - Syntax highlighting clearly visible
- **Toggle button**: The WYSIWYG/Source toggle should be visible and Source should be the active state
- **Benefit**: This isn't a dumbed-down tool — power users get full raw access
- **Layout**: Same format with overlay bar

## Screenshot Implementation Approach

Screenshots should be generated by the `scripts/generate_store_assets.mjs` script using Playwright. The approach:

1. **Build the extension**: `npm run build`
2. **Launch Chrome with the extension loaded** (Playwright can load unpacked extensions)
3. **Prepare sample Markdown files** in a temp directory (create realistic content matching each screenshot concept)
4. **Navigate the extension UI** programmatically:
   - Open the folder containing sample files
   - Click on the target file in the sidebar
   - For screenshot 4: toggle to source mode
5. **Capture screenshot** at exactly 1280×800

**Fallback approach** (if programmatic extension control is too fragile):
- Manually open the extension, navigate to each state, and capture screenshots
- Use Playwright only for the overlay generation: create an HTML page that composites the raw screenshot with the overlay title bar, then screenshot that composite

**Overlay Template** (for all screenshots):
```html
<!-- 1280×800 composite -->
<div style="width:1280px; height:800px; position:relative;">
  <!-- Raw screenshot as background -->
  <img src="raw_screenshot.png" style="width:100%; height:100%; object-fit:cover;">
  <!-- Overlay bar at top -->
  <div style="position:absolute; top:0; left:0; right:0; 
              background: linear-gradient(180deg, rgba(15,23,42,0.92) 0%, rgba(15,23,42,0.75) 100%);
              padding: 24px 40px; color:white; font-family:'Outfit',sans-serif;">
    <div style="font-size:28px; font-weight:700; margin-bottom:4px;">Overlay Title</div>
    <div style="font-size:16px; font-weight:400; color:#94a3b8;">Overlay subtitle</div>
  </div>
</div>
```
